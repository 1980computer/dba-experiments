Day 45: Framer Impressionist

Canvas, GetImageData, Utils.random

I took Henrique Gusso's very cool prototype and made a few modifications. I wanted to create a filter to make a photo look like an Impressionist painting. The effect is constraint by the browser's ability to handle effect changes on the number of elements. It was a fun project, and lots of learnings from Henrique's source code. Safari only. Running on Chrome will likely cause it to crash.

Day 44: Accordion

JSON.parse Utils.domLoadDataSync, .join and Scroll

Having seen Elliot Noltens's great demo of an accordion list view, I decided to give it a try myself. I learned from his prototype a great way to add a system of vetted UI gradients to the prototype via JSON. For some reason scroll.updateContent() doesn't work with state changes, so I added an invisible layer to extend the scroll.

Day 42: Gooey Morph

Working with Contrast and Blur

I stumpled upon this great blog article by Rohn, and decided to give the gooey morph effect a shot. Instead of injecting the blur and contrast as strings in the style properties as Rohn suggested, Framer already accepts constrast and blur as direct inputs. I tried it, and it worked. Swipe left to view effect. This only works in Safari.
